### Root cause

The vulnerability stemmed from the `ruby-jwt` library allowing empty or nil HMAC keys during JWT verification. ### Impact

An attacker could forge JWTs signed with HMAC algorithms (e.g., HS256, HS384, HS512) without knowing the actual secret key. By providing an empty key, the attacker could create a valid signature for any payload, leading to authentication bypass, unauthorized access, or other impacts depending on how the JWTs are used in the application.
