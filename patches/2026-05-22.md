## GHSA-q8mj-m7cp-5q26 · qs

2026-05-22 · JavaScript · **MODERATE** 5.3 · Pattern: `NULL_DEREF→CRASH`

### Root cause

The `qs.stringify` function, when configured with `encodeValuesOnly` and `comma` array format, would attempt to call an `encoder` function on `null` or `undefined` array elements. The default encoder expects a string or number, leading to a `TypeError` when `null` or `undefined` is passed, causing the application to crash.

### Impact

An attacker could provide input that includes arrays with `null` or `undefined` values, triggering a `TypeError` in the `qs.stringify` function. This would lead to a denial of service for any application using the affected `qs` library version with the specific configuration, making the application unavailable.

### Fix summary

The patch modifies the `maybeMap` utility call within the `stringify` function. It now explicitly checks if an array element is `null` or `undefined` before attempting to apply the `encoder` function. If the value is `null` or `undefined`, it is passed through unchanged, preventing the `TypeError`.
