## GHSA-3jmg-p96m-m328 · github.com/gtsteffaniak/filebrowser

2026-05-19 · Go · **HIGH** 0.0 · Pattern: `INFO_DISCLOSURE→DIRECTORY_LISTING`

### Root cause

The application's default settings for anonymous users were not properly restricted, allowing them to inherit certain settings and potentially access information that should have been protected. Specifically, the `ApplyUserDefaults` function did not explicitly prevent anonymous users from having their scopes modified or from inheriting other sensitive settings.

### Impact

An unauthenticated attacker could potentially gain access to shared information or other configuration details that were not intended for public exposure, leading to unauthorized information disclosure.

### Fix summary

The patch introduces an explicit check at the beginning of the `ApplyUserDefaults` function to immediately return if the user is 'anonymous', preventing any default settings from being applied to them. Additionally, a condition related to `u.Username != "anonymous"` when checking `len(u.Scopes) == 0` was removed, ensuring that scope application logic is consistent and not bypassed for anonymous users in other parts of the code.

## GHSA-9q9q-324x-93r2 · bandit

2026-05-19 · Erlang · **HIGH** 0.0 · Pattern: `DOS→RESOURCE_EXHAUSTION`

### Root cause

The vulnerability stemmed from an unbounded read in the chunked transfer encoding parser. An attacker could send a `Transfer-Encoding: chunked` request with an extremely large chunk size header, which the server would attempt to parse entirely into memory before processing the actual chunk data. This could lead to excessive memory allocation.

### Impact

An unauthenticated attacker could send a specially crafted HTTP request to the server, causing it to consume a large amount of memory and potentially crash, leading to a denial of service for legitimate users.

### Fix summary
