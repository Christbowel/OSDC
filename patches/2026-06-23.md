## GHSA-pwx3-qcgw-vh7h · gogs.io/gogs

2026-06-23 · Go · **HIGH** 8.8 · Pattern: `CSRF→STATE_CHANGE`

### Root cause

The application was vulnerable to Cross-Site Request Forgery (CSRF) because it did not implement sufficient CSRF protection mechanisms for sensitive actions, specifically those related to changing an organization owner. An attacker could craft a malicious webpage that, when visited by an logged-in administrator, would trigger a request to change the organization owner without the administrator's explicit consent.

### Impact

An attacker could trick an organization owner into unknowingly transferring ownership of an organization to an attacker-controlled account, leading to a full takeover of the organization.

## GHSA-wv27-2vqp-j7g5 · gogs.io/gogs

2026-06-23 · Go · **HIGH** 8.1 · Pattern: `SSRF→INTERNAL_ACCESS`

### Root cause

The application allowed users to specify a local file path as a mirror address for a repository without proper authorization checks. The `ParseRemoteAddr` function, which was responsible for validating and parsing remote addresses, did not consistently enforce the `CanImportLocal()` permission check for all call sites, specifically when setting a mirror address via the repository settings page. This allowed any authenticated user to import a local repository, effectively reading local files.

### Impact

An attacker could import local files from the server's filesystem into a Gogs repository, leading to information disclosure. This could potentially expose sensitive configuration files, user data, or other arbitrary files accessible by the Gogs process.
