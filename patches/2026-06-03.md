## GHSA-c27g-q93r-2cwf · vite

2026-06-03 · JavaScript · **HIGH** 0.0 · Pattern: `UNSANITIZED_INPUT→COMMAND`

### Root cause

The application was vulnerable to command injection on Windows due to insufficient sanitization of user-provided file names when launching an editor. The `childProcess.spawn` function, when used with `cmd.exe /C`, did not properly escape special characters in the file path, allowing an attacker to inject arbitrary commands.

### Impact

An attacker could execute arbitrary commands on the system where the `launch-editor` package is used, leading to Remote Code Execution (RCE).

### Fix summary

The patch removes the previous file name pattern validation and instead implements a more robust escaping mechanism for command-line arguments on Windows. It now uses `childProcess.exec` with `shell: true` and escapes special characters like `&`, `<`, `>`, `|`, `%`, and `^` using `^` before executing the command. It also handles double quoting for paths containing spaces or escaped characters.
