### Root cause

The vulnerability stemmed from a weak 32-bit hashing algorithm used to generate cache keys for HTTP responses in the `HttpTransferCache`. This weak hashing led to a high probability of hash collisions, meaning different HTTP requests could generate the same cache key. When a collision occurred, the response for one request could overwrite or be served in place of another, leading to cross-request data leakage or state poisoning.

### Impact

An attacker could potentially craft requests that collide with legitimate requests, causing sensitive data from one user's request to be served to another user, or poisoning the cache with malicious data that affects subsequent legitimate requests. ### Root cause

The HtmlSanitizer component in Symfony did not properly sanitize URLs embedded in certain HTML attributes, specifically the 'content' attribute of <meta http-equiv="refresh"> tags and several other URL-related attributes like 'data', 'codebase', 'archive', and 'longdesc'. The existing UrlAttributeSanitizer was not configured to handle these specific attributes, allowing malicious URLs to bypass sanitization.

### Impact

An attacker could inject arbitrary, unsanitized URLs into the HTML output, potentially leading to Cross-Site Scripting (XSS) attacks, open redirects, or other client-side vulnerabilities if the sanitized HTML is rendered in a browser.
