## GHSA-6hq5-7373-42rg · phpoffice/phpspreadsheet

2026-07-23 · PHP · **HIGH** 7.7 · Pattern: `SSRF→INTERNAL_ACCESS`

### Root cause

The application's `WEBSERVICE()` function allowed fetching content from URLs but did not explicitly disable HTTP redirects. This meant that even if the initial URL was whitelisted, a malicious server could redirect the request to an internal or otherwise restricted resource, bypassing the intended domain whitelist.

### Impact

An attacker could bypass the domain whitelist and potentially access internal network resources, cloud metadata, or other sensitive information that should not be publicly exposed.

### Key diff

```
--- a/src/PhpSpreadsheet/Calculation/Web/Service.php

             ],
         ];
```

### Fix summary

The patch addresses the vulnerability by explicitly setting `follow_location` to `0` in the HTTP context options for `file_get_contents`. This prevents the `WEBSERVICE()` function from automatically following HTTP redirects, thereby enforcing the domain whitelist for the initial request.

## GHSA-2mrg-gjxq-2gvr · phpoffice/phpspreadsheet

2026-07-23 · PHP · **HIGH** 7.5 · Pattern: `DOS→RESOURCE_EXHAUSTION`

### Root cause

The Gnumeric reader did not limit the size of decompressed gzip data, allowing an attacker to provide a specially crafted compressed file that would expand to an extremely large size, consuming all available memory. Additionally, the OLERead component lacked checks for cyclic dependencies in its block chains, which could lead to infinite loops when processing malformed OLE files.

### Impact

An attacker could cause a denial of service by exhausting system memory or CPU resources, making the application unresponsive or crashing it, by providing a malicious Gnumeric or OLE file.

The Gnumeric reader now limits the `gzdecode` operation based on a calculated maximum length derived from PHP's memory_limit. The OLERead component now includes a loop detection mechanism that throws an exception if a cyclic dependency is found in the block chains, preventing infinite loops.

Additionally, it adds provider ID checks to several cookie signing and verification functions to prevent cross-provider cookie misuse.
