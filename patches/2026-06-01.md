## GHSA-qjwp-hrq6-r26r · kas

2026-06-01 · Python · **LOW** 0.0 · Pattern: `UNSANITIZED_INPUT→COMMAND`

### Root cause

The application directly used user-supplied input (git branch/ref names) in a 'git checkout' command without properly distinguishing between a branch name and a commit SHA. This allowed a specially crafted SHA-like string to be interpreted as a valid commit, even if it wasn't, potentially leading to unexpected behavior or command injection if not handled carefully.

### Impact

An attacker could trick the application into checking out an arbitrary commit by providing a SHA-like string that is not a valid branch, potentially leading to the checkout of unintended code or a denial of service if the 'git checkout' command fails.

### Key diff

```

+++ b/kas/repos.py
```

### Fix summary

The patch modifies the `checkout_cmd` function to explicitly append `^{commit}` to the desired reference if it matches a 40 or 64 character hexadecimal SHA pattern. This forces Git to interpret the string as a commit object, preventing it from being mistaken for a branch name or other Git reference.
