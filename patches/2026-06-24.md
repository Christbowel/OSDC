## GHSA-7fq5-7wr8-rjwj · github.com/OliveTin/OliveTin

2026-06-24 · Go · **HIGH** 7.5 · Pattern: `RACE_CONDITION→TOCTOU`

### Root cause

The application used a single global template instance (`tpl`) for parsing templates concurrently across multiple requests. The `text/template` library in Go is not designed to handle concurrent `Parse` calls on the same template instance, leading to a race condition where template definitions from one request could interfere with or contaminate another.

### Impact

An attacker could potentially achieve cross-request command contamination, meaning that template data or commands intended for one user's request could be inadvertently processed or executed in the context of another user's request, leading to information disclosure or unauthorized actions.

### Key diff

```
-	t, err := tpl.Parse(source)
+	clone, err := tpl.Clone()
+	if err != nil {
+		return "", err
+	}
+
+	t, err := clone.Parse(source)
```

### Fix summary

The patch addresses the race condition by cloning the root template instance (`tpl`) before calling `Parse`. This ensures that each template parsing operation uses a unique, isolated template instance, preventing concurrent modifications and cross-request contamination.

## GHSA-prj9-97mp-mwh2 · github.com/OliveTin/OliveTin

2026-06-24 · Go · **MODERATE** 4.3 · Pattern: `UNSANITIZED_INPUT→COMMAND`

### Root cause

The application allowed user-defined action arguments to use the 'ot_' prefix, which was intended for internal system arguments. These 'ot_'-prefixed arguments were not subject to the same validation and filtering as other user-defined arguments, allowing an attacker to inject arbitrary values into system-controlled variables that were later used in shell commands.

### Impact

An attacker could inject malicious commands into the 'ot_executionTrackingId' or 'ot_username' system arguments, leading to arbitrary command execution on the server where OliveTin is running.

+	return ok

### Fix summary

The patch introduces a validation step to prevent user-defined action arguments from using the reserved 'ot_' prefix. It also ensures that all system-injected arguments, including those used in 'ShellAfterCompleted' commands, are properly validated against predefined types before being used.
