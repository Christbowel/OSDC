## GHSA-8whx-365g-h9vv · loofah

2026-07-21 · Ruby · **LOW** 0.0 · Pattern: `UNSANITIZED_INPUT→XSS`

### Root cause

The `allowed_uri?` method in Loofah did not correctly handle `javascript:` URIs that were split by HTML5 named whitespace character references (e.g., `&Tab;`, `&NewLine;`). While it unescaped HTML entities and stripped control characters, it missed these specific named character references, allowing an attacker to bypass the URI validation.

### Impact

An attacker could craft a malicious URI using named whitespace character references to bypass Loofah's sanitization, potentially leading to Cross-Site Scripting (XSS) by executing arbitrary JavaScript in the user's browser.

### Fix summary

The patch introduces a new regular expression `WHITESPACE_CHARACTER_REFERENCES` to explicitly detect and remove HTML5 named whitespace character references like `&Tab;` and `&NewLine;` from URI strings. This ensures that `javascript:` schemes obfuscated with these references are correctly identified and blocked during URI validation.
