## GHSA-55f6-pf8r-c2f4 · openbabel

2026-07-06 · Python · **HIGH** 7.8 · Pattern: `BUFFER_OVERFLOW→HEAP`

### Root cause

The application did not properly validate the number of translation vectors read from input files (MOPAC, MSI, Gaussian formats). It would increment `numTranslationVectors` and use it as an index into the `translationVectors` array without checking if the index exceeded the array's allocated size, leading to writes beyond the buffer boundary.

### Impact

An attacker could craft a malicious input file that, when parsed by Open Babel, would cause an out-of-bounds write. This could lead to a denial of service (crash), arbitrary code execution, or other unpredictable behavior due to memory corruption.

### Key diff

```
--- a/src/formats/mopacformat.cpp
+++ b/src/formats/mopacformat.cpp
@@ -167,7 +167,8 @@ namespace OpenBabel
                 y = atof((char*)vs[3].c_str());
                 z = atof((char*)vs[4].c_str());
 
-                translationVectors[numTranslationVectors++].Set(x, y, z);
+                if (numTranslationVectors < 3)
+                  translationVectors[numTranslationVectors++].Set(x, y, z);
```

### Fix summary

The patch adds explicit boundary checks before writing to the `translationVectors` array. Specifically, it checks `if (numTranslationVectors < 3)` before incrementing `numTranslationVectors` and setting the vector, ensuring that no more than three translation vectors are ever written, which is the expected maximum size of the array.
