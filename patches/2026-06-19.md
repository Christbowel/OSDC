## GHSA-vmhf-c436-hxj4 · jupyterlab

2026-06-19 · Python · **MODERATE** 0.0 · Pattern: `UNSANITIZED_INPUT→XSS`

### Root cause

The JupyterLab extension manager displayed package metadata, including the 'homepage_url', without proper sanitization or validation of the URI protocol. An attacker could inject arbitrary JavaScript by crafting a malicious 'homepage_url' with a 'javascript:' protocol.

### Impact

An attacker could execute arbitrary JavaScript in the context of the user's browser when they view the malicious extension's details in the extension manager, potentially leading to session hijacking, data theft, or further client-side attacks.

### Fix summary

The patch introduces a new function, `isProtocolAllowed`, which explicitly checks if a URL's protocol is either 'http:' or 'https:'. This function is then used to validate the `entry.homepage_url` before it is rendered as an anchor tag's `href` attribute, preventing the use of malicious protocols like 'javascript:'.
