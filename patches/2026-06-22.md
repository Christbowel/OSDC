## GHSA-5pm9-r2m8-rcmj · paymenter/paymenter

2026-06-22 · PHP · **CRITICAL** 9.9 · Pattern: `UNCLASSIFIED`

The application allowed users to upload files via the EasyMDE editor in ticket creation and viewing forms. The `completeUpload` method in Livewire components directly stored these uploaded files without sufficient validation of their content or type, allowing an attacker to upload malicious executable files.

## GHSA-3qq3-668m-v9mj · gogs.io/gogs

2026-06-22 · Go · **MODERATE** 4.9 · Pattern: `DOS→RESOURCE_EXHAUSTION`

### Root cause

The application was vulnerable to a Denial of Service due to how it handled Git tree entries. Without the `Verbatim: true` option, the `LsTree` and `Entries` functions in the underlying Git library could be forced to perform excessive processing, leading to resource exhaustion when encountering specially crafted Git objects.

### Impact

An attacker could craft a malicious Git repository or wiki content that, when viewed or processed by Gogs, would consume excessive CPU and memory resources, leading to a denial of service for legitimate users.

### Key diff

```
- dir, err := gitRepo.LsTree(entry.ID().String())
+ dir, err := gitRepo.LsTree(entry.ID().String(), git.LsTreeOptions{Verbatim: true})
- entries, err := dir.Entries()
+ entries, err := dir.Entries(git.LsTreeOptions{Verbatim: true})
```

### Fix summary

The patch addresses the vulnerability by consistently adding `git.LsTreeOptions{Verbatim: true}` to all calls to `gitRepo.LsTree` and `commit.Entries`. This option ensures that Git tree entries are processed verbatim, preventing the underlying Git library from performing expensive normalization or processing that could be exploited for resource exhaustion.

## GHSA-w6j9-vw59-27wv · gogs.io/gogs

2026-06-22 · Go · **HIGH** 0.0 · Pattern: `MISSING_AUTH→ENDPOINT`

### Root cause

The Gogs application allowed authentication via a reverse proxy header without validating if the request originated from a trusted proxy. An attacker could forge the `ReverseProxyAuthenticationHeader` in a direct request to Gogs, bypassing the intended security measure that assumes the header is set only by a trusted upstream proxy.

### Impact

An attacker could impersonate any user, including administrators, by sending a crafted HTTP request with the appropriate authentication header, leading to full compromise of the Gogs instance.
