## GHSA-375f-4r2h-f99j ┬╖ bandit

2026-05-07 ┬╖ Erlang ┬╖ **MODERATE** 0.0 ┬╖ Pattern: `INSECURE_DEFAULTтЖТCONFIG`

### Root cause

The Bandit web server library incorrectly determined the URI scheme (HTTP/HTTPS) by trusting the client-supplied request target, even when the connection itself was plaintext. This allowed an attacker to trick the server into believing a plaintext connection was secure (HTTPS) if they manipulated the request target.

### Impact

An attacker could potentially bypass security controls that rely on the `Plug.Conn.scheme` value, such as secure cookie flags or HSTS, by making the server believe a non-secure connection was secure. This could lead to information disclosure or other security misconfigurations.

### Fix summary

The patch removes the `request_target` parameter from the `determine_scheme/2` function. The scheme is now solely determined by whether the underlying transport layer is secure or not, preventing client-supplied values from influencing this critical security decision.
