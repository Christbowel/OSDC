## GHSA-4q5v-7g7x-j79w · compliance-trestle

2026-05-28 · Python · **HIGH** 8.4 · Pattern: `PATH_TRAVERSAL→FILE_WRITE`

### Root cause

The vulnerability existed because the application recursively rendered Jinja templates, allowing an attacker to inject malicious Jinja expressions into the template output. This recursive rendering, combined with the ability to control parts of the template content, enabled path traversal to write files to arbitrary locations.

### Impact

An attacker could achieve arbitrary file write on the system where the application is running, potentially leading to remote code execution or data corruption.
