## GHSA-pqwm-q9pv-ph8r · shivammathur/setup-php

2026-05-20 · YAML · **MODERATE** 5.6 · Pattern: `UNSANITIZED_INPUT→COMMAND`

### Root cause

The application was vulnerable to command injection because user-supplied input for PHP versions, extensions, and other configuration values was directly concatenated into shell commands without proper escaping. This allowed an attacker to inject arbitrary shell commands.

### Impact

An attacker could execute arbitrary commands on the runner machine where the GitHub Action is executed, potentially leading to full compromise of the environment or exfiltration of sensitive data.

## GHSA-468c-vq7p-gh64 · plug

2026-05-20 · Erlang · **HIGH** 0.0 · Pattern: `DOS→RESOURCE_EXHAUSTION`

### Root cause

The `Plug.Conn.read_part_headers` function, responsible for parsing multipart request headers, did not properly enforce the `:length` option. This allowed an attacker to send a multipart request with an arbitrarily large header block, causing the server to accumulate an unbounded amount of data in memory.

### Impact

An attacker could send a specially crafted multipart request to a Plug-based application, leading to excessive memory consumption and ultimately a denial of service for the application.
