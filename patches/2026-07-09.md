### Root cause

## GHSA-rqrh-8wpv-x7hh · github.com/enchant97/note-mark/backend

2026-07-09 · Go · **HIGH** 0.0 · Pattern: `PATH_TRAVERSAL→FILE_WRITE`

### Root cause

The application used unsanitized user-controlled input (username, book slug, note slug) directly in file path construction during the data export process. This allowed an attacker to inject path traversal sequences (e.g., `../`) to access or create files outside the intended export directory.

### Impact

An attacker could write arbitrary files to arbitrary locations on the server's file system, potentially leading to remote code execution, data corruption, or denial of service.

### Key diff

```
- bookDir := path.Join(exportDir, user.Username, book.Slug)
+ bookSlug := filepath.Base(book.Slug)
+ userName := filepath.Base(user.Username)
+ bookDir := path.Join(exportDir, userName, bookSlug)
```

### Fix summary

The patch introduces `filepath.Base()` calls on all user-controlled path components (username, book slug, note slug) to strip any directory separators, ensuring that only the base name is used. Additionally, it strengthens the regular expressions for username and slug validation in the database types to explicitly anchor the patterns, preventing path traversal sequences from being stored.
