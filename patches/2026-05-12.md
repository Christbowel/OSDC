## GHSA-c73c-x77g-854r · @gitlawb/openclaude

2026-05-12 · JavaScript · **MODERATE** 6.5 · Pattern: `CSRF→STATE_CHANGE`

### Root cause

The original code processed OAuth callback parameters in a way that allowed an 'error' parameter to bypass the crucial 'state' parameter validation. If an 'error' parameter was present, the state check was skipped, enabling an attacker to cancel an ongoing OAuth flow for a victim.

### Impact

An attacker could perform a Denial of Service (DoS) by sending a specially crafted OAuth callback URL with an 'error' parameter to a victim, causing their authentication flow to be prematurely terminated and preventing them from logging in or completing the OAuth process.

### Key diff

```
-            const code = parsed.searchParams.get('code')
-            const state = parsed.searchParams.get('state')
-            const error = parsed.searchParams.get('error')
+            const result = validateOAuthCallbackParams(
+              {
+                code: parsed.searchParams.get('code'),
+                state: parsed.searchParams.get('state'),
+                error: parsed.searchParams.get('error'),
```

### Fix summary

The patch introduces a new `validateOAuthCallbackParams` function that centralizes and strictly enforces the order of validation. It ensures that the 'state' parameter is always validated first, regardless of other parameters like 'error', preventing the bypass. It also refactors the handling of callback parameters in both manual and HTTP server contexts to use this new validation logic.

## GHSA-c38f-wx89-p2xg · ujson

2026-05-12 · Python · **HIGH** 0.0 · Pattern: `DOS→RESOURCE_EXHAUSTION`

### Root cause

The vulnerability existed because the `ujson.dump()` function, when writing to a file, did not properly deallocate Python objects (`string`) if an error occurred during the `PyTuple_Pack` call or the subsequent `write` operation to the file. This led to a reference count not being decremented, causing a memory leak.

### Impact

An attacker could potentially cause a denial of service by repeatedly triggering write failures, leading to the application consuming excessive memory and eventually crashing or becoming unresponsive.

### Key diff

```
--- a/src/ujson/python/objToJSON.c
+++ b/src/ujson/python/objToJSON.c
@@ -929,6 +934,7 @@ PyObject* objToJSONFile(PyObject* self, PyObject *args, PyObject *kwargs)
   if (argtuple == NULL)
   {
     Py_XDECREF(write);
+    Py_DECREF(string);
     return NULL;
   }
 
@@ -936,6 +942,7 @@ PyObject* objToJSONFile(PyObject* self, PyObject *args, PyObject *kwargs)
   if (write_result == NULL)
   {
     Py_XDECREF(write);
+    Py_DECREF(string);
     Py_XDECREF(argtuple);
     return NULL;
   }
```

### Fix summary

The patch addresses the memory leak by adding `Py_DECREF(string)` calls in error handling paths. Specifically, if `PyTuple_Pack` fails after `objToJSON` has returned a string, or if the `write` operation itself fails, the `string` object's reference count is now correctly decremented, preventing the leak.

## GHSA-m77w-p5jj-xmhg · openclaude

2026-05-12 · JavaScript · **CRITICAL** 0.0 · Pattern: `PRIVILEGE_ESCALATION→ROLE`

### Root cause

The `dangerouslyDisableSandbox` parameter in the BashTool was exposed to the AI model. Although the prompt attempted to guide the model on when to use it, the model could directly set this parameter in its tool calls, bypassing the intended user approval flow and security restrictions.

### Impact

An attacker, via a malicious AI model, could execute arbitrary commands outside the sandbox environment, leading to potential remote code execution, data exfiltration, or system compromise on the host running the OpenClaude application.

### Fix summary

The patch removes `dangerouslyDisableSandbox` from the model-facing schema of the BashTool and PowerShellTool. It introduces an internal-only `_dangerouslyDisableSandboxApproved` flag, which must be true for the sandbox to be disabled. This ensures that sandbox disabling can only be triggered by trusted internal logic, not directly by the AI model's input.
