## GHSA-37h2-6p4f-mp3q · serena-agent

2026-07-08 · Python · **HIGH** 8.3 · Pattern: `SSRF→INTERNAL_ACCESS`

### Root cause

The Flask dashboard was accessible on a fixed port without proper host validation. This allowed an attacker to use DNS rebinding to trick a victim's browser into making requests to the local dashboard, bypassing same-origin policy and enabling memory poisoning leading to RCE.

### Impact

An attacker could achieve remote code execution (RCE) on the victim's machine by exploiting the DNS rebinding vulnerability to manipulate the local Flask dashboard's memory.

### Fix summary

The patch adds a `before_request` hook to the Flask application. This hook verifies that the `request.host` header matches either '127.0.0.1:{port}' or 'localhost:{port}', aborting the request with a 403 Forbidden error if it does not, thereby preventing DNS rebinding attacks.

## GHSA-q95x-7g78-rccv · oneringbuf

2026-07-08 · RUST · **MODERATE** 0.0 · Pattern: `USE_AFTER_FREE→MEMORY`

### Root cause

The `IntoRef` trait previously included an `into_ref` method that took `self` by value. This allowed the underlying buffer to be dropped (freed) while a reference to it (DroppableRef or NonDroppableRef) was still held, leading to a use-after-free condition when the reference was later accessed.

### Impact

An attacker could potentially cause memory corruption, leading to denial of service, arbitrary code execution, or information disclosure by exploiting the freed memory.

### Key diff
