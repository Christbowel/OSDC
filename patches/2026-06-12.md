## GHSA-98xf-r82g-9mhx · @langchain/langgraph-checkpoint-mongodb

2026-06-12 · JavaScript · **MODERATE** 6.7 · Pattern: `UNSANITIZED_INPUT→NOSQL`

### Root cause

The LangGraph MongoDBSaver directly used user-controlled input from `config.configurable` fields (like `thread_id`, `checkpoint_ns`, `checkpoint_id`) in MongoDB queries without proper type validation or sanitization. This allowed an attacker to inject MongoDB query operators (e.g., `$ne`, `$gt`) into these fields, altering the intended query logic.

### Impact

An attacker could bypass access controls and retrieve or modify checkpoint data belonging to other tenants or threads by manipulating the query parameters, leading to cross-tenant state access and data leakage or corruption.

### Fix summary

The patch introduces a `getStringConfigValue` helper function to explicitly validate that configurable values are strings. This function is applied to all user-controlled fields (`thread_id`, `checkpoint_ns`, `checkpoint_id`) before they are used in MongoDB queries, preventing the injection of NoSQL operators.
